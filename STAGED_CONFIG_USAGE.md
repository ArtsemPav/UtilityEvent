# Работа с большими конфигами (Staged Config)

## Описание

При загрузке больших JSON-файлов (>1 события для LiveEvent или >1 ConfigSet для SinglePick) редактор автоматически переходит в режим **staged config** — исходный файл сохраняется целиком, но редактируется только один выбранный элемент.

## Как это работает

### LiveEvent (вкладка ✏️Редактор LiveEvent)

1. **Загрузка большого файла:**
   - Загрузите JSON с несколькими событиями через "📂 Загрузить JSON"
   - Появится сообщение: "📦 Загружен большой конфиг ... — N событий"
   - Отобразится выпадающий список с EventID всех событий

2. **Выбор события для редактирования:**
   - Выберите нужное событие из списка
   - Нажмите "✏️ Открыть"
   - В редактор загрузится только это событие

3. **Редактирование:**
   - Работайте с событием как обычно (сегменты, ноды, награды)
   - Изменения пока только в редакторе

4. **Применение изменений:**
   - Нажмите "💾 Применить изменения в исходный конфиг"
   - Отредактированное событие заменит оригинал в исходном файле
   - Можно выбрать другое событие и продолжить редактирование

5. **Валидация:**
   - Кнопка "✅ Проверить валидацию" проверяет **весь исходный конфиг** с применёнными изменениями

6. **Экспорт (вкладка 💾 Экспорт LiveEvent):**
   - Экспортируется **весь исходный конфиг** с применёнными изменениями
   - Предпросмотр позволяет выбрать отдельное событие или весь конфиг

### SinglePick (вкладка 🎰Редактор SinglePick)

Работает аналогично LiveEvent:

1. **Загрузка:** JSON с >1 ConfigSet → выбор из списка
2. **Редактирование:** Открывается один ConfigSet
3. **Применение:** "💾 Применить изменения в исходный конфиг"
4. **Валидация:** Проверяет весь исходный конфиг
5. **Экспорт:** Весь конфиг с изменениями

## Преимущества

- **Экономия памяти:** В редакторе только один элемент, а не весь файл
- **Безопасность:** Исходный файл не теряется при редактировании
- **Гибкость:** Можно редактировать несколько элементов по очереди
- **Валидация:** Проверяется целостность всего конфига, а не только редактируемого элемента

## Важные моменты

- Если загружен файл с 1 событием/ConfigSet — staged режим **не активируется**, работа как обычно
- Кнопка "🆕 Новый конфиг" сбрасывает staged режим
- При загрузке нового файла — предыдущий staged конфиг заменяется
- Изменения применяются только после нажатия "💾 Применить изменения"
